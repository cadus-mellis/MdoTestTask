Тестовое задание для МДО
https://mdo.ru/

Задание:
(C# .NET Framework):
Написать web-сервис из трех функций:
- Функция подключения к БД MS SQL Server;
- Функция, которая возвращает версию MS SQL;
- Функция, которая закрывает подключение к БД.

Написать UI-приложение которое может подключаться к вышеупомянутому сервису и вызывать его методы.

Задание максимально открытое и специально не формализовано.
В результате мы хотим понимать образ мыслей кандидата, умение продумывать интерфейсы взаимодействия внутри проекта, логику отдельных компонентов и т.д.
Если данные мысли будут изложены доступно в документе, будет считаться бонусом.

# Архитектурные решения
Так как задание требует три отдельные операции - открыть соединение, получить версию SQL Server и закрыть соединение - реализовал API через connection session с `connectionId`.  
Контракты API вынес в отдельный проект `SqlVersionService.Contracts`, чтобы переиспользовать их и в web-сервисе, и в desktop-клиенте без дублирования.  
Прикладную логику, инфраструктуру и web-слой разделил по отдельным проектам. Для Web API 2 на .NET Framework использовал Autofac в качестве DI, также добавил Swagger, Serilog, таймаут сессий и автоочистку зависших соединений.

# Проекты решения

## SqlVersionService.Contracts
Общие request/response модели для web-сервиса и desktop-клиента.

## SqlVersionService.Application
Основная логика работы с connection session: открытие соединения, получение версии SQL Server, закрытие, таймаут и очистка просроченных сессий.

## SqlVersionService.Infrastructure
Инфраструктурные реализации, в том числе in-memory реестр активных соединений.

## SqlVersionService.Web
ASP.NET Web API 2 сервис: контроллеры, DI, Swagger, логирование, обработка ошибок и запуск фоновой очистки.

## SqlVersionService.Desktop
WinForms-клиент для вызова методов сервиса: открыть соединение, получить версию SQL Server и закрыть соединение.

# Ограничение текущего решения
В текущей реализации активные соединения хранятся в памяти приложения, но в production нужно было бы сохранять состояние в БД.
